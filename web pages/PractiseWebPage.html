18th

在最近几天我听人家说eclipse不错，于是就去看看想要下载一个试一下，后来不知道为什么就开始看VS Code，之后就下载了一个。 我先是将这个安装在我的沙盒里面，其实当时好像主要还是想要试一试虚拟机，因为用这个不用担心主电脑受损，所以本着试一试的态度 想下载eclipse，偶然间看到一个文章上面显示VS Code更受欢迎，所以下了一个VS Code试一试。

用了一下之后就感觉这个东西真的不错，首先内存真的很少，运行起来也就占几百兆的内存而已，Intellij Idea要上G，而且还很慢， 再加上我本来电脑性能就不是很好，所以啊还是用了这个更轻量级的VS Code。之后这个东西也是可以支持各种语言的，因为这个本来就 不是专门的IDE，而是文本编辑器，所以更加的全能。我本身也没有什么东西特别依赖Intellij Idea，所以我轻松地从Idea转到了VS Code。

28th

今天我有下载了一个新的写程序的软件，名字叫做NotePad++，这个东西可以说就只是NotePad的增强版，非常的简洁，甚至几乎是可以当做 NotePad来用还毫无违和感，而且支持很多的语言，我叫得上名字的交不上名字的都有。不过缺点当然还是有的，我感觉真的只能当做一个NotePad来用了，连 跑程序我都不知道怎么跑，我唯一会写的编程语言就是Python和一点点java，java我的办法就是用命令行javac之后再java这样跑，Python就直接用IDLE直接跑 也差不多，都是不能直接在这个软件上面跑，要到外界找自己编程语言原生的东西来跑，不过也还行，毕竟这样非常快。

不过吧，虽然功能性上与 VS Code感觉差得远，但是这个玩意儿要快得多了去了，就跟NotePad一样，平时还可以当NotePad用很方便，VS Code相对来说就慢得多了

我发现一个 规律，我是用的这些编程软件一直在朝简洁的方向前进，一开始是JetBrains的Pycharm，后来要写java就换成了IDEA + 插件，发现太卡了，电脑配置真的不行， 后来就用上了VS Code，之后就上了NotePad++，估计到最后我可能就直接连编程软件都懒得要了，直接NotePad走起，完全不用配置，一切靠手撸，但是我觉得 我才不会这么傻，语法高亮和代码补全是写程序的人最后的底线，绝对不能抛弃。

package Exercise4;

public class Mix4 {
    int counter = 0;
    public static void main(String[] args) {
        int count = 0;
        Mix4 [] m4a = new Mix4[20];
        int x = 0;
        while (x < 19) {
            m4a[x] = new Mix4();
            m4a[x].counter = m4a[x].counter + 1;
            count = count + 1;
            count = count + m4a[x].maybeNew(x);
            x = x + 1;
        }
        System.out.println(count + " " + m4a[1].counter);
    }

    public int maybeNew(int index) {
        if (index < 1) {
            Mix4 m4 = new Mix4();
            m4.counter = m4.counter + 1;
            return 1;
        }
        return 0;
    }
}
